# 🔎 שלב 0 – Risk Classification (קריטי)
 
סווג את הפרומפט לפי רמת פעולה בעולם האמיתי:
 
| רמה | סוג |
|------|------|
| R0 | מייצר טקסט בלבד (Guidelines / Generator) |
| R1 | קורא מידע בלבד (Read-only tools) |
| R2 | מפעיל כלים כותבים (write/update/delete) |
| R3 | Agent אוטונומי / execution shell |
 
קבע Risk Level לפני המשך בדיקה.
 
⚠ אם R2 או R3 → רגישות אבטחה גבוהה.
⚠ אם R0 → אין להחמיר יתר על המידה בנושאי tool execution.

# 🛡 שלב 3 – אבטחה (מחולקת ל-3 תתי ציונים)
 
במקום ציון אבטחה אחד, תן:
 
### A. Injection Resistance (1-5)
- התייחסות להוראות זדוניות?
- סימון UNTRUSTED?
- הפרדת תוכן משתמש מהוראות?
 
### B. Execution Risk (1-5)
- האם מתבצעת פעולה מערכתית?
- האם יש אישור לפני write?
- האם קיימת Least Privilege?
- אם R0 → ציון זה לא רלוונטי, קבע 5 כברירת מחדל.
 
### C. Data Exposure (1-5)
- חשיפת credentials?
- גישה למידע רגיש?
- export אוטומטי?
 
חשב ממוצע 3 הסעיפים → Security Score.
 
⚠ אל תסמן CRITICAL אם אין Execution בפועל.

# 🚨 שלב 4 – זיהוי דפוסים מסוכנים
 
בדוק דפוסי Injection אך:
- הבחנה בין טקסט דוגמה לבין יכולת בפועל
- אל תעניש על Role words אם הם חלק מהגדרת תפקיד לגיטימית
 
רמות סיכון:
 
CRITICAL:
- ביצוע פעולה מערכתית ללא אישור
- חשיפת מידע רגיש בפועל
- עקיפת הרשאות
 
HIGH:
- כלי write ללא guardrails
- חסר מוחלט בהפרדת תוכן משתמש
 
MEDIUM:
- הגנות חלקיות
- חוסר ב-UNTRUSTED tagging
 
LOW:
- שיפורים קלים
 
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 
# 🧮 שלב 5 – חישוב
 
Quality Score = סכום 6 קטגוריות (max 30)
 
Security Score = ממוצע (Injection + Execution + Data Exposure)
 
Final Decision Logic:
 
APPROVED:
- Quality ≥ 24
- Security ≥ 4
- אין CRITICAL
 
NEEDS_REVISION:
- Quality 18-23
- או Security בין 3-4
 
REJECTED:
- Quality < 18
- Security < 3
- CRITICAL סיכון אמיתי (Execution/Data Exposure)
 
⚠ אם Risk Level = R0 → אל תדחה רק בגלל Injection חלקי.
